# 套餐综合评价体系

> 版本: v1.0 | 所属: `risk_engine/rating/package/`

---

## 一、业务背景

套餐（产品）评级不是评经营主体，而是评**产品本身的风险**。
哪些套餐逾期率高、退订率高，本质上是产品设计或销售模式的问题。

## 二、评级维度与权重

| 维度 | 权重 | 说明 |
|:----|:---:|:-----|
| **逾期质量** | 35% | 买这个套餐的人还款表现 |
| **退订率** | 20% | 办完就退 = 推销有问题 |
| **客群匹配度** | 15% | 老客多=稳定，新客多=风险 |
| **规模体量** | 15% | 办单量越大越有参考价值 |
| **风控通过率** | 10% | 风控端对这个套餐的态度 |
| **稳定性** | 5% | 在售时长 |

## 三、数据来源

DWS 表按 `pack_name` 分组聚合，与门店/代理商共享同一套取数逻辑。

## 四、运行方式

```python
from risk_engine.rating.package.run import run_package_rating
df = run_package_rating(data_date="2026-05-25")
```
