# Frontend — 前端网站

> **设计思想：** 前端是平台的界面，负责数据可视化和用户交互。
> 核心原则：**只渲染不计算**、**状态集中管理**、**组件可复用**。

## 页面结构

```
frontend/
├── realtime_dashboard/     # 实时大盘 — TPS、拒绝率、模型调用量
├── case_analysis/          # 案例回溯 — 输入特征 + 决策结果详情
├── model_ops/              # 模型监控 — PSI、分数分布、AUC 趋势
├── rule_management/        # 规则配置 — 低代码化规则管理
└── report_center/          # 报表中心 — 历史报表查看/下载
```

## 设计原则

### 1. 前端只渲染，不下放业务逻辑
- PSI、AUC、逾期率等指标全部由后端算好再返回
- 前端只负责展示图表和表格，不做任何聚合计算

### 2. 状态集中管理
- 全局状态通过 `stores/` 统一管理（Pinia / Redux）
- 避免 props drilling，组件只关心自己的展示逻辑

### 3. 图表组件可复用
- `components/charts/` 封装通用图表（折线图/柱状图/热力图/饼图）
- 统一基于 ECharts 或 AntV，通过 props 传入配置和数据

### 4. 后端 API 调用
- `services/` 层统一管理 API 请求
- 自动处理 token 注入、错误重试、loading 状态
